# Progress Clarification Checklist

Use this checklist when reviewing `progress_update.md`.

## Must be clear before reporting

- Update date
- Status period covered
- Completed tasks or deliverables
- Completion dates when available
- In-progress tasks
- Blocked or delayed tasks
- Owners of blockers and next actions
- Risks or issues that need reporting
- Decisions made since last update
- Any items that actually require the change/replan workflow

## Do not treat as progress-only if

- Scope changed
- Deadline changed
- Budget changed
- New deliverable added
- Task sequence or dependency changed materially
- Owner requests a new plan snapshot

Those items belong in the change/replan lane.
