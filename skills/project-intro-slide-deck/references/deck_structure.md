# Project Introduction Deck Reference

Use this reference when creating or reviewing a project introduction deck generated from `project_plan.md`, `gantt_tasks.csv`, and a PDF in `company_style/`.

## Audience

Default audience: company owner, project owner, sponsor, client stakeholder, or internal decision-maker.

## Inputs

- `project_plan.md`: source of project facts, scope, deliverables, risks, roles, decisions, and next steps.
- `gantt_tasks.csv`: source of task timing, owners, dependencies, work packages, and timeline detail.
- `company_style/*.pdf`: source of company name, brand cues, color palette, layout style, tone, logos, and visual reference.

## Tone

Professional, practical, and clear. The deck should explain what the project is, why it matters, what will be delivered, how the timeline looks, who is involved, what risks exist, and what decisions are still needed.

## Core slides

1. Title / project introduction
2. Why this project matters
3. Scope and deliverables
4. Success criteria
5. Timeline overview
6. Key milestones
7. Workstreams and task summary
8. Roles and responsibilities
9. Risks and mitigation
10. Open decisions
11. Next steps

## Style use from company_style PDF

Use the PDF as a reference for:

- company name
- color palette
- visual style
- logo placement
- typography feel
- tone and terminology
- footer/header style

Do not treat the style PDF as project-plan content unless the user specifically asks.

## What to avoid

- Do not bundle or reuse a fixed script in the skill.
- Do not paste the full project plan into slides.
- Do not hide missing dates, owners, or approvals.
- Do not invent project facts to make the deck look complete.
- Do not create a timeline that appears final when it is based on assumptions.
